# ============================================================
# EPIC: EP05 - Gestión de Presupuestos, Pagos e Indicadores
# Proyecto: TrackOps - AutoSync
# US relacionadas: US09, US13, US14, US18, US19, US20, US48
# ============================================================

Feature: Gestión de Presupuestos, Pagos e Indicadores Financieros
  Como parte del sistema TrackOps
  Quiero gestionar presupuestos, pagos e indicadores del taller
  Para controlar el desempeño financiero y brindar transparencia al cliente

  # ============================================================
  # US09 - Registrar avance de reparación
  # ============================================================

  Background:
    Given el administrador ha iniciado sesión en TrackOps
    And existe una orden de trabajo con código "ODT-001" registrada en el sistema

  Scenario: Técnico registra un avance de reparación
    Given existe una reparación en proceso para la orden "ODT-001"
    And el técnico "Juan Pérez" está asignado
    When el técnico registra el siguiente avance:
      | campo       | valor                         |
      | descripcion | Se cambió el filtro de aceite |
      | porcentaje  | 50                            |
      | fecha       | 2026-07-01                    |
    Then el sistema debe almacenar el avance registrado
    And debe actualizar el porcentaje de progreso de la orden a "50%"

  Scenario Outline: Registrar avances con distintos porcentajes
    Given existe una reparación activa en la orden "<orden>"
    When el técnico registra un avance con porcentaje "<porcentaje>"
    Then el sistema debe actualizar el progreso a "<porcentaje>%"

    Examples:
      | orden   | porcentaje |
      | ODT-001 | 25         |
      | ODT-002 | 50         |
      | ODT-003 | 75         |
      | ODT-004 | 100        |

  Scenario: Técnico intenta registrar avance en orden cancelada
    Given existe una orden "ODT-005" con estado "Cancelado"
    When el técnico intenta registrar un avance
    Then el sistema debe mostrar el mensaje "No se puede registrar avance en una orden cancelada"

  # ============================================================
  # US13 - Generar presupuesto
  # ============================================================

  Scenario: Administrador genera un presupuesto exitosamente
    Given el administrador accede a la orden "ODT-001"
    When ingresa los costos del servicio con los siguientes datos:
      | concepto            | cantidad | precio_unitario |
      | Mano de obra        | 2        | 80.00           |
      | Filtro de aceite    | 1        | 35.00           |
      | Aceite sintético 4L | 1        | 120.00          |
    And confirma la generación del presupuesto
    Then el sistema debe calcular el total en S/. 315.00
    And debe generar el presupuesto con estado "Pendiente de aprobación"
    And debe notificar al cliente que tiene un presupuesto disponible

  Scenario: Intentar generar presupuesto sin conceptos
    Given el administrador accede a la orden "ODT-001"
    When intenta generar un presupuesto sin ingresar ningún concepto
    Then el sistema debe mostrar el mensaje "Debe ingresar al menos un concepto para generar el presupuesto"

  Scenario Outline: Generar presupuestos con distintos montos
    Given existe una orden "<orden>" con servicio "<servicio>"
    When el administrador genera un presupuesto con monto total de S/. "<monto>"
    Then el sistema debe registrar el presupuesto con total S/. "<monto>"
    And el estado del presupuesto debe ser "Pendiente de aprobación"

    Examples:
      | orden   | servicio                  | monto  |
      | ODT-001 | Cambio de aceite          | 315.00 |
      | ODT-002 | Revisión de frenos        | 480.00 |
      | ODT-003 | Reparación de transmisión | 950.00 |
      | ODT-004 | Diagnóstico eléctrico     | 200.00 |

  # ============================================================
  # US14 - Aprobar presupuesto
  # ============================================================

  Scenario: Cliente aprueba un presupuesto exitosamente
    Given el cliente "Carlos Ríos" ha iniciado sesión
    And existe un presupuesto generado para su vehículo "ABC-123" con total S/. 315.00
    When el cliente revisa el presupuesto y selecciona "Aprobar"
    Then el sistema debe registrar la aprobación
    And debe cambiar el estado a "Aprobado"
    And debe notificar al taller que puede iniciar la reparación

  Scenario: Cliente rechaza un presupuesto con motivo
    Given el cliente "Carlos Ríos" ha iniciado sesión
    And existe un presupuesto con total S/. 950.00 para su vehículo
    When el cliente selecciona "Rechazar" e ingresa el motivo "Monto elevado"
    Then el sistema debe registrar el rechazo con el motivo indicado
    And debe cambiar el estado a "Rechazado"
    And debe notificar al taller del rechazo

  Scenario Outline: Clientes aprueban o rechazan presupuestos
    Given el cliente "<cliente>" tiene un presupuesto de S/. "<monto>" para su vehículo "<placa>"
    When el cliente selecciona "<decision>"
    Then el sistema debe cambiar el estado del presupuesto a "<estado_resultado>"

    Examples:
      | cliente     | monto  | placa   | decision | estado_resultado |
      | Carlos Ríos | 315.00 | ABC-123 | Aprobar  | Aprobado         |
      | María López | 480.00 | XYZ-456 | Aprobar  | Aprobado         |
      | Pedro Soto  | 950.00 | LMN-789 | Rechazar | Rechazado        |
      | Ana Flores  | 200.00 | QRS-321 | Aprobar  | Aprobado         |

  # ============================================================
  # US19 - Registrar pago
  # ============================================================

  Scenario: Administrador registra un pago exitosamente
    Given existe una orden "ODT-001" con estado "Completado" y presupuesto aprobado de S/. 315.00
    When el administrador registra el pago con los siguientes datos:
      | campo       | valor      |
      | monto       | 315.00     |
      | metodo_pago | Tarjeta    |
      | fecha_pago  | 2026-07-02 |
      | referencia  | TXN-98765  |
    Then el sistema debe almacenar el pago correctamente
    And debe cambiar el estado de la orden a "Pagado"
    And debe habilitar la emisión de factura

  Scenario Outline: Registrar pagos con diferentes métodos
    Given existe una orden "<orden>" pendiente de pago con monto S/. "<monto>"
    When el administrador registra el pago con método "<metodo>"
    Then el sistema debe guardar el pago con método "<metodo>"
    And el estado de la orden debe actualizarse a "Pagado"

    Examples:
      | orden   | monto  | metodo        |
      | ODT-001 | 315.00 | Efectivo      |
      | ODT-002 | 480.00 | Tarjeta       |
      | ODT-003 | 950.00 | Transferencia |
      | ODT-004 | 200.00 | Yape          |

  Scenario: Registrar pago con monto cero
    Given existe una orden "ODT-001" con total S/. 315.00
    When el administrador intenta registrar un pago de S/. 0.00
    Then el sistema debe mostrar el mensaje "El monto del pago debe ser mayor a cero"
    And no debe registrarse el pago

  # US20 - Emitir factura
  # ============================================================

  Scenario: Sistema emite factura tras pago registrado
    Given la orden "ODT-001" tiene un pago registrado de S/. 315.00
    And el cliente "Carlos Ríos" tiene su DNI registrado
    When el administrador solicita la emisión de la factura
    Then el sistema debe generar la factura con los siguientes datos:
      | campo    | valor            |
      | cliente  | Carlos Ríos      |
      | monto    | 315.00           |
      | servicio | Cambio de aceite |
      | estado   | Emitida          |
    And debe enviar la factura al correo del cliente

  Scenario: Intentar emitir factura sin pago registrado
    Given existe una orden "ODT-002" sin pago registrado
    When el administrador intenta emitir la factura
    Then el sistema debe mostrar el mensaje "No se puede emitir factura sin un pago registrado"

  Scenario Outline: Emitir facturas para distintos clientes
    Given la orden "<orden>" tiene pago registrado de S/. "<monto>"
    When se solicita la emisión de factura para el cliente "<cliente>"
    Then el sistema debe generar la factura por S/. "<monto>" para "<cliente>"
    And el estado de la factura debe ser "Emitida"

    Examples:
      | orden   | monto  | cliente     |
      | ODT-001 | 315.00 | Carlos Ríos |
      | ODT-002 | 480.00 | María López |
      | ODT-003 | 950.00 | Pedro Soto  |
      | ODT-004 | 200.00 | Ana Flores  |

  # US18 - Generar reporte de ingresos
  # ============================================================

  Scenario: Administrador genera reporte de ingresos por período
    Given existen pagos registrados en el sistema entre "2026-06-01" y "2026-06-30"
    When el administrador selecciona el período "Junio 2026" y solicita el reporte
    Then el sistema debe mostrar el total de ingresos del período
    And debe desglosar los ingresos por tipo de servicio

  Scenario Outline: Generar reportes de ingresos por diferentes períodos
    Given existen pagos registrados en el período "<periodo>"
    When el administrador solicita el reporte de ingresos para "<periodo>"
    Then el sistema debe mostrar el total de S/. "<total>" para ese período

    Examples:
      | periodo       | total    |
      | Abril 2026    | 4500.00  |
      | Mayo 2026     | 6200.00  |
      | Junio 2026    | 3800.00  |
      | Julio 2026    | 1200.00  |

  # ============================================================

  Scenario: Administrador visualiza indicadores del taller
    Given existen datos registrados en el sistema
    When el administrador accede al panel principal de indicadores
    Then el sistema debe mostrar:
      | indicador                  | tipo    |
      | Órdenes completadas        | Número  |
      | Ingresos del mes           | Monto   |
      | Técnico con más trabajos   | Nombre  |
      | Tiempo promedio de servicio| Días    |

  Scenario: Panel de indicadores sin datos registrados
    Given no existen datos registrados en el sistema aún
    When el administrador accede al panel de indicadores
    Then el sistema debe mostrar el mensaje "No hay datos suficientes para mostrar indicadores"
